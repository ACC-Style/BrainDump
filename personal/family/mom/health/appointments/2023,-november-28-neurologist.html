Milestones Checklist

This checklist is roughly established by the breakdown from Outline of Tickets for SPAs in Vue

  • Project Initiation: Conduct a kick-off meeting involving representatives from Product Lines, UX Team, Frontend Developers, and Backend Developers to align on objectives and expectations. Define project scope and set up project tracking.
  • Requirement Gathering: Gather and document the design and functional requirements for the SPA. Set a milestone to finalize the requirements.
  • Technical Feasibility: Conduct a technical feasibility study based on the gathered design and requirements. Review and adjust project scope if necessary.
  • [ ] Analytics engagement: outline goals and metrics that are needed for antibiotics. Engage the analytics team and fold any technical requirements for analytics into the project from the beginning.
  • Design Phase: Follow-up on the design after the technical feasibility study, making necessary iterations and adjustments. Define a milestone for finalizing the design.
  • Development - Basic Application Setup: Define and set up the data structure and API calls, construct the page layout, create the code branch, and set up page access. Also, set up the Vue APP SHELL and initiate the application. Set a milestone for the completion of this basic application setup.
  • Development - Component Construction: Create multiple tickets for each component that needs to be developed or modified. Set a milestone for the completion of all components.
  • Development - Feature Specifics: Create multiple tickets for each application-specific feature that needs to be scaffolded. Set a milestone for the completion of all features.
  • Integration and Testing: Integrate components with the application and refine features in cooperation with QA. Set a milestone for the completion of integration and testing.
  • User Acceptance Testing: Conduct user acceptance testing to ensure that the application meets the defined requirements and user expectations. Set a milestone for the completion of UAT.
  • Beta Focus Group (Optional): If applicable, conduct a beta test with a focus group. Use their feedback to make final adjustments before the full launch.
  • Launch Preparation: Prepare for the application launch, ensuring all systems are ready and all required parties are informed.
  • Launch and Post-Launch Review: Launch the SPA, monitor its performance, gather feedback, and conduct a post-launch review. Set a milestone for the completion of the post-launch review.
