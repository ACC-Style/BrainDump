Backlog Ticket Structure:

  1. Title: A concise, descriptive title for the feature or issue
  2. Description:
    • Background: Brief context or background information.
    • Objective: Clear statement of what the ticket aims to achieve.
    • User Story: Format - As a [type of user], I want [an action] so that [a value or result].
  3. Acceptance Criteria:
    • List of conditions that must be met for the ticket to be considered complete.
    • Should be specific, measurable, achievable, relevant, and time-bound (SMART).
  4. Dependencies: Any dependencies or prerequisites for the ticket.
  5. Checklist for Components:
    • Component Definition: Define the zone, providing a screenshot, requirements from an editor's perspective, and the URL location.
    • Component Identification: IT to identify and isolate the component.
    • Design Implementation: A front-end designer/developer to create an HTML prototype of the component using the Arches design system.
    • Group Grooming: Conduct a group grooming session to identify any additional tickets or tasks necessary to complete the story.
