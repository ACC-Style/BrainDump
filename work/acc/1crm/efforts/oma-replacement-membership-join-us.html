Select Application Type

He has never dictated the need for the UI but he recognizes a great divide between what is shown in demo and what is currently in place in our solution.

The Good

From the quick demo I just received the dropdown is driving the options the user is eligible for which is the requirement.

What is lacking from the demo solution

The select application is a simple dropdown with only the application names that the user is eligible for. This has some lacking experiences from what is currently built which are.

  • A description of the differences of member types
  • Direct Comparison of Options
  • Pricing of the options before you click
  • A quick checklist of details you will need to fill in after you start.

This last item is what concerns JP the most because of how this impact's user experience and education before an application is kicked off.

Compounding the Problem

This solution has compounding problems because of how the application selection doesn't inform the user on what information they should collect before they start and the experience of saving and returning to the application.
