Script Glossary & Categorization

--------------------------------------------------------- 

SPRINT A - Define the Batch 

--------------------------------------------------------- 

  1. Select conversion candidate for the batch and define scope
  2. Run an Audit Query against the code base of Sitecore within scope for code components & plan needed conversions
  3. Run an Audit Query against the CMS Content Areas (RichText & CustomScripts) within scope for Editorial Generated UI & start planning for remediation
  4. Groom 

--------------------------------------------------------- 

SPRINT B - Component Conversions

--------------------------------------------------------- 

  1. Convert the components 
  2. Deploy to lower environments DEV/SPR 
  3. QA test the converted components 
  4. Promote to PROD 
  5. List of bulk changes - Mapping of changes 

--------------------------------------------------------- 

SPRINT C - Editorial Created Remediation 

--------------------------------------------------------- 

  1. SYNC SPR 
  2. Execute bulk conversion script with a small sample of pages in DEV/SPR - Using Mapping 
  3. Clean up rich text via script (S&D) 
  4. Review effected pages 
  5. Create jira tickets for content issues identified during review 
  6. Work with BL to resolve found issues 
  7. Promote to RLS 
  8. Demo with BL and obtain sign-off 

--------------------------------------------------------- 

SPRINT D - Batch Conversion

--------------------------------------------------------- 

  1. Create PRD/Master/Web DBs backup for potential rollback 
  2. Run bulk conversion script in Author only with auto-publishing turned off 
  3. IT performs spot checks 
  4. BL reviews and validates pages, with support from UI, DEV, and QA teams 
  5. Garbage Clean up of Empty Content areas and RichText
  6. Resync to lower environments?

--------------------------------------------------------- 

SPRINT Z - Done! 

--------------------------------------------------------- 

  1. Garbage cleanup
