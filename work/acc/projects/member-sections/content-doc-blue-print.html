Content & Page Blocks Document

This document outlines the evergreen and changeable content blocks used across various user states for Member Section pages. The structure follows the naming convention: [USER STATE].[Page].[Optional Sub Page] and supports modularity and reusability.

This document is used for:

  • Mapping content for the components used across the different user states within the member section.

Component Reference Tags.
These tags are used to explain how or what needs to be collected for this block of content.

  • Evergreen - Any content block labeled evergreen is global and it content is not changeable.
    • Dynamic Defining dynamic placeholders within evergreen content will use this the format [[LABEL NAME]] for example [[MEMBER SECTION CHAIR]]
  • Data Model - Some content will just be place holder or an excel sheet to collect the data within so it can flooded into the final UI. Names, Locations, Emails etc. depending on the UI.
  • Changeable - This content has placeholder content based on our prototype and is indicative copy intended to be changed by the Member Section. There might be notes with the content of max word count or other playbook rules that we would like the final content to follow.

Note: These different content collection Tags for content areas will be labeled inline throughout this document where applicable. Member Section leads should focus only on updating the changeable areas with the provided indicative copy as a guide.

It is intended to be distributed to Member Section leads for content collection and review.
