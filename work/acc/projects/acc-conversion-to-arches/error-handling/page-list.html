Page List

1. 404 - Page Not Found

  • Solution: Provide a custom 404 page that clearly explains the page is not found, includes a search bar, and links to popular or important pages like the homepage or contact page.
  • Tip: Add humorous or lighthearted content to ease frustration.

2. 400 - Bad Request

(Redirect to 404)

  • Solution: Display a clear message explaining the request was incorrect or malformed. Suggest actions the user can take, such as reloading the page or checking for errors in the URL.
  • Tip: Guide users to report the issue if it persists.

3. 401 - Unauthorized

(Sanitize for Security)

  • Solution: Redirect users to a login page with a clear explanation that access requires authentication. Offer a link to reset passwords if needed.
  • Tip: Keep the login process simple and user-friendly, avoiding unnecessary steps.

4. 403 - Forbidden

(Suppress for Security)

  • Solution: Inform the user that they don’t have permission to access the requested page, and suggest contacting support if they believe this is an error.
  • Tip: If the user is logged out, consider prompting them to log in, if applicable.

5. 500 - Internal Server Error

(Sanitize for Security)

  • Solution: Display a friendly error message stating that something went wrong on the server. Provide a link to the homepage or suggest the user try again later.
  • Tip: Include an option to report the issue directly to the support team.

6. 502 - Bad Gateway

(Sanitize for Security generic 500)

  • Solution: Explain that the server encountered an issue while acting as a gateway. Inform users this is likely a temporary issue and suggest trying again later.
  • Tip: Offer a status page or contact option in case the problem persists.

7. 503 - Service Unavailable

(Sanitize for Security generic 500 to 503)

  • Solution: Notify users that the service is temporarily unavailable due to server overload or maintenance. Include an estimated time for when the service will be back online, if possible.
  • Tip: Add a subscription or notification option so users can be alerted when the site is back up.

8. 504 - Gateway Timeout

(Sanitize or Suppress for Security generic 500)

  • Solution: Display a message indicating the server took too long to respond, and suggest refreshing the page or trying again later.
  • Tip: Offer a link to check the site's status or provide a more interactive error page (e.g., FAQ section) that users can explore while waiting.

9. 405 - Method Not Allowed

(Suppress for Security only available non production)

  • Solution: Inform users that the request method they used is not allowed. Clearly state what methods are allowed (e.g., GET, POST) and offer navigation back to valid pages.
  • Tip: Offer a clear explanation or auto-redirect to the homepage or main section of the site.

10. 408 - Request Timeout

  • Solution: Show users a message indicating their connection was too slow or the request took too long. Encourage them to check their internet connection or retry the action.
  • Tip: Add a "Retry" button that automatically resubmits the request.

11. 429 - Too Many Requests

*(Sanitize)

  • Solution: Politely inform users they’ve made too many requests in a short time and suggest waiting a few minutes before trying again.
  • Tip: Include a countdown or time estimate for when they can try again.

12. 410 - Gone

  • Solution: Clearly state that the requested page has been permanently removed and won’t be available again. Offer a search bar or links to other similar content.
  • Tip: If the content has been moved, provide the new link or page.

13. 415 - Unsupported Media Type

(Suppress and handle in UI)

  • Solution: Inform the user that the media format they tried to upload or access isn’t supported. Provide information on the types of media that are accepted.
  • Tip: Offer a link to a help or FAQ page explaining how to format files for uploading.

14. 501 - Not Implemented

(Suppress)

  • Solution: Notify users that the requested functionality is not supported. Provide a suggestion for alternative actions or contact support if they require the missing feature.
  • Tip: If it’s a planned feature, provide an option for users to sign up for updates on its development.

General Tips for Error Pages:

  • Make It Easy to Navigate Away: Always include a clear path back to the homepage, a search bar, or popular sections so users can easily recover.
  • Human-Friendly Language: Avoid using technical jargon in error messages. Use conversational language that’s easy to understand.
  • Maintain Branding: Keep the error pages visually consistent with your site’s overall design and tone to maintain a cohesive experience.
  • Track Errors for Improvement: Log and monitor these error codes to identify recurring issues and improve site performance over time.

Here’s a list of critical error pages that should be suppressed or sanitized to improve security protection:

Sanitized:

These errors can still be shown to the user but should be cleaned up to avoid exposing sensitive information.

  1. 500 - Internal Server Error
  2. 400 - Bad Request
  3. 502 - Bad Gateway
  4. 503 - Service Unavailable
  5. 504 - Gateway Timeout
  6. 429 - Too Many Requests

Suppressed:

These errors should be replaced with generic messages to avoid disclosing details about access restrictions, authentication, or supported methods.

  1. 403 - Forbidden
  2. 401 - Unauthorized
  3. 405 - Method Not Allowed
